# v0.1.1

Removed the Vulkan render path.

- **`src/ffplay_renderer.c`** / **`src/ffplay_renderer.h`**: deleted entirely.
- **`src/ffplay.c`**:
    - Inlined SDL-only definitions from the deleted header (`VIDEO_BACKGROUND_TILE_SIZE`, `enum VideoBackgroundType`, `RenderParams`).
    - Removed `enable_vulkan`, `vulkan_params`, `vk_renderer` and all branches (display, `do_exit`, window events, main).
    - `video_image_display`: SDL texture path only.
    - `configure_video_filters`: `buffersink` colorspace constraints now always applied (previously only when `!vk_renderer` — same effective behaviour now).
    - `create_hwaccel`: no longer depends on Vulkan; calls `av_hwdevice_ctx_create` directly.
    - Option table: removed `-enable_vulkan`, `-vulkan_params`.
    - `main`: `SDL_CreateRenderer` direct (no `SDL_WINDOW_VULKAN` / `vk_renderer_create`).

## Notes

1. The linked FFmpeg DLLs still carry `--enable-vulkan` and `--enable-libplacebo` (Gyan's build). Only the exe source changed — the Vulkan/libplacebo display pipeline is no longer used.
2. `-hwaccel` (`create_hwaccel`) no longer requires a Vulkan renderer to be active.
3. Whether the decoder can use Vulkan internally is determined by `-hwaccel` + `av_hwdevice_ctx_create`.
